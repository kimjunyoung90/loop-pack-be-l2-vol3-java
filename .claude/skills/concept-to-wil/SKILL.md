# Concept to WIL Skill

사용자가 모르는 개념에 대해 질문하면, 아래 흐름을 따른다.

```
1. 개념 설명
2. WIL 문서에 저장
```

---

## 1. 개념 설명

사용자의 질문에 대해 먼저 충분히 설명한다.

### 설명 원칙

- 핵심을 먼저, 세부 사항은 나중에
- 비유나 대비를 활용하여 이해하기 쉽게
- 코드 예시가 도움이 되면 반드시 포함
- 사용자의 현재 작업 맥락과 연결하여 설명

---

## 2. WIL 문서에 저장

설명이 끝나면, 해당 내용을 옵시디언 WIL 문서에 자동으로 추가한다.

### 저장 위치 결정

1. `프로젝트/루프팩/` 하위의 가장 최근 주차 폴더를 확인한다.
2. 해당 주차의 `03. WIL.md` 파일에 append 한다.

### 저장 형식

```markdown

---

## {개념 제목}

{설명 본문}

- 핵심 개념 정의
- 코드 예시 (있는 경우)
- 비교 표 (있는 경우)
```

### 저장 규칙

- 기존 WIL 내용과 중복되는 주제가 있으면 저장하지 않고, 사용자에게 "이미 WIL에 정리된 내용입니다"라고 알린다.
- 저장 완료 후 사용자에게 "WIL에 추가했습니다"라고 간단히 알린다.
- 구어체나 대화체는 제거하고, 정리된 문어체로 작성한다.

---

## 스킬 적용 판단 기준

이 스킬은 **작업 중 발생하는 개념 질문**에만 적용한다.

### 적용 O

- ":: 문법이 뭐지?"
- "toList()랑 collect 차이가 뭐야?"
- "dirty checking이 무슨 뜻이야?"
- "guard() 메서드가 뭐하는 거야?"

### 적용 X

- 프로젝트 구조나 코드 위치에 대한 질문 ("이 파일 어디 있어?")
- 작업 지시 ("리뷰해줘", "수정해줘")
- 요구사항 분석 요청
